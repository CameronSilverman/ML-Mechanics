# ML Maker Studio — Frontend

Visual drag-and-drop workspace for building ML pipelines, with an interactive learning section.

This README was partially generated by an LLM

## Setup

```bash
cd client
npm install
npm start       # http://localhost:3000
```

## Pages

| Route | Page | Description |
|-------|------|-------------|
| `/` | `HomePage` | Landing page with links to workspace and learn section |
| `/whiteboard` | `WhiteboardPage` | Full sandbox canvas with save/load/export |
| `/learn` | `LearnPage` | Lesson catalogue grouped by category |
| `/learn/:lessonId` | `LessonPage` | Guided lesson with instruction panel + restricted canvas |

## Usage

### Whiteboard

- **Add blocks** — drag from sidebar → drop on canvas
- **Move blocks** — drag blocks around the canvas
- **Pan canvas** — click and drag on the canvas background
- **Recenter** — click the crosshair button (bottom-left of canvas)
- **Connect blocks** — drag from an output port (right edge) to an input port (left edge)
- **Delete a wire** — click it
- **Remove all wires on a port** — right-click the port
- **Edit properties** — right-click a block → Edit Properties
- **Delete a block** — right-click → Delete
- **View code** — toolbar Code button or `Ctrl+K`
- **Save / Load** — requires an account; `Ctrl+S` / `Ctrl+O`
- **Export / Import** — JSON file via toolbar or `Ctrl+I`

### Learn

- Browse lessons on `/learn`, click a card to start
- The **Lesson** tab contains rich instructional content
- The **Steps** tab shows the current task; each step validates automatically as you build
- Hints are available per step on demand
- Some lessons lock specific block properties and training settings
- Use **Open in Workspace** to continue editing freely in the full sandbox

## Adding a Lesson

Edit `src/data/lessons.js` — push one object to `LESSONS`. The file's JSDoc describes every field. No other file changes are needed.

## Adding a Component

Edit `src/data/mlComponents.js` — add an entry to the appropriate category object. Then add a matching `case` in `utils/codeGenerator.js` to emit the correct Keras call, and optionally in `utils/pipelineWarnings.js` for property validation.
