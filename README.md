# ML Maker Studio

A visual drag-and-drop environment for building machine learning pipelines — no code required.

This README was partially generated by an LLM

## Features

- **Visual canvas** — infinite panning workspace with drag-and-drop ML blocks, bezier wire connections, and a recenter button
- **Component library** — sidebar with categorised blocks: Data, Structure, Layers, Activation, and Output
- **Code generation** — export your pipeline as executable Keras/TensorFlow Python code (Functional API)
- **Training settings bar** — persistent inline controls for optimizer, learning rate, loss function, epochs, batch size, and test split
- **Pipeline diagnostics** — real-time error/warning/info panel that checks for shape mismatches, disconnected ports, and invalid settings
- **Learn section** — interactive lessons with rich content (text, callouts, code, images), step-by-step validation, locked properties, and hint system
- **Project management** — save, load, rename, and delete cloud projects (requires account); export/import via JSON; built-in starter templates
- **Authentication** — JWT-based sign-up/login with tokens stored in localStorage

## Project Structure

```

  src/
    components/     UI components (Canvas, Sidebar, panels, modals)

    styles/         index.css — all styles

  routes/         auth.js, projects.js
  middleware/     auth.js (JWT verification)

  index.js        Entry point
```

## Prerequisites

- **Node.js** ≥ 18
- **PostgreSQL** ≥ 14

## Setup

### 1. Database

```bash
createdb ml_maker_studio
```

### 2. Server

```bash
cd server
cp .env.example .env       # edit with your DB credentials and a JWT secret
npm install
npm run dev                 # starts on http://localhost:5000
```

Tables (`users`, `projects`) are created automatically on first run.

### 3. Client

```bash
cd client
npm install
npm start                   # starts on http://localhost:3000
```

Navigate to `/whiteboard` for the canvas or `/learn` for interactive lessons.

## API

All endpoints are prefixed with `/api`. Project routes require a `Bearer` token in the `Authorization` header.

| Method | Endpoint | Auth | Description |
|--------|----------|------|-------------|
| POST | `/api/auth/register` | No | Create account |
| POST | `/api/auth/login` | No | Sign in |
| GET | `/api/projects` | Yes | List user's projects (metadata only) |
| GET | `/api/projects/:id` | Yes | Get project with full canvas data |
| POST | `/api/projects` | Yes | Create a project |
| PUT | `/api/projects/:id` | Yes | Update name and/or canvas data |
| DELETE | `/api/projects/:id` | Yes | Delete a project |
| GET | `/api/health` | No | Server health check |

### Auth shapes

```json
// POST /api/auth/register  or  /api/auth/login
// Request
{ "email": "user@example.com", "password": "secret123" }

// Response (201 register / 200 login)
{
  "token": "eyJhbG...",
  "user": { "id": 1, "email": "user@example.com", "createdAt": "..." }
}
```

### Project shapes

```json
// POST /api/projects
{ "name": "My CNN", "data": { "blocks": [...], "connections": [...], "trainingSettings": {...} } }

// PUT /api/projects/:id  (all fields optional; omitting data preserves existing)
{ "name": "Renamed" }
{ "name": "Renamed", "data": { "blocks": [...], "connections": [...] } }
```

## Adding Lessons

Open `client/src/data/lessons.js` and push a new object to the `LESSONS` array. No other file needs to change. See the JSDoc at the top of that file for the full lesson schema, including rich content block types (`text`, `heading`, `callout`, `code`, `image`, `video`, `divider`), step definitions with inline `validate` functions, and the solution shape used by the solution checker.

## Adding Components

Open `client/src/data/mlComponents.js` and add an entry to the appropriate category. Each component needs a `type`, `label`, `icon`, `color`, `defaults`, optional `propertyDefs` (for dropdown fields), and `inputs`/`outputs` port arrays. The code generator (`utils/codeGenerator.js`) and pipeline warnings checker (`utils/pipelineWarnings.js`) may also need a corresponding `case` for the new type.
